Custom IP port (e.g. 8080) only affects the IP address. A panel hostname always uses normal web ports 80 / 443. Sidebar links stay on whatever host:port you used to open the panel (relative URLs).

2. Panel SSL (HTTPS)

Issues a Let's Encrypt certificate for the panel hostname. DNS A must point here, and port 80 must be open for the hostname (Let's Encrypt does not use the custom IP port like 8080). Test first: http://your-hostname/ should open this panel.

3. Access by IP

Independently of the hostname: allow opening the panel with the server IP (useful for recovery). You can also use a non-standard public port.

Default 80 → http://{{ s.server_ip }}/. Example 2087 → http://{{ s.server_ip }}:2087/. (Internal app stays on localhost:{{ s.app_port }}.)

Tip: disable IP access only after hostname + SSL work. Keep SSH for recovery.

4. Browser security

Turn on only when you open the panel over HTTPS. Breaks login on plain HTTP (IP).

After changing cookies / session options run: sudo systemctl restart srv-panel
